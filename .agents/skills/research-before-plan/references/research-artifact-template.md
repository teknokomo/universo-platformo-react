# Research Artifact Template

Use this template for source-backed research documents created before or during PLAN mode.

```markdown
# Research: <Topic>

> Created: YYYY-MM-DD
> Status: Draft | Reviewed | Superseded
> Trigger: <user command or request>
> Follow-up plan: ../plan/<planned-plan-file>.md

## Research Question

State the external question and the repository decision this research supports.

## Source Inventory

| Source | Type | Date / Freshness | Why It Matters |
|--------|------|------------------|----------------|
| <URL> | Primary / Secondary / Tertiary | <date or freshness note> | <decision relevance> |

## Key Findings

- Separate facts from inferences.
- Include concise evidence-backed findings only.

## Conflicts And Uncertainty

- Note sources that disagree.
- Note weak evidence, stale information, or missing documentation.

## Project Implications

- Explain what the findings mean for this repository.
- Identify affected files, rules, packages, workflows, or Memory Bank areas.

## Recommended Decision

State the recommended path and why it follows from the evidence.

## Open Questions Before PLAN

- List only questions that materially affect planning.
- If there are none, write "None."

## Sources

- <URL>
```

Research artifacts must be written in English, following Memory Bank rules.
