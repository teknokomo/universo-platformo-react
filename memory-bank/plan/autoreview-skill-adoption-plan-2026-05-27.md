## Skill Shape Decision

-   Use the skill name `autoreview`.
-   Place it at `.agents/skills/autoreview/`.
-   Treat it as an imported external skill that is vendored and adapted for this
    repository.
-   Do not create `.codex/skills`, `.claude/skills`, `.kiro/skills`, symlinks, or
    external install instructions.
-   Keep `SKILL.md` concise and move implementation detail into the script. No
    `README.md`, `INSTALLATION_GUIDE.md`, or `CHANGELOG.md` inside the skill.
-   Preserve MIT attribution in `.agents/skills/SOURCES.md` and either add a
    short copyright/SPDX note to copied scripts or otherwise ensure the upstream
    MIT notice is included with the copied material.

## Potential Challenges

-   The upstream helper calls external CLIs (`codex`, `claude`, `droid`,
    `copilot`) that may not all exist in this environment. The default path should
    remain Codex, and other engines should be opt-in.
-   Running a real review can be slow and may consume model/tool budget. The
    implementation validation should use `--help`, `--dry-run`, syntax checks, and
    skill validation unless the user explicitly requests live review execution.
-   The upstream `SKILL.md` contains useful process guidance mixed with
    OpenClaw-specific operations. Implementation should rewrite the skill body
    rather than copy it verbatim.
-   The copied Python script is large. Keep edits surgical and validate behavior
    with syntax/help/dry-run checks before attempting live engine runs.
-   The helper builds bundles from untracked files. In this repository, agents
    must still preserve unrelated user changes and avoid treating all dirty
    worktree files as authored by the current task.
-   MIT attribution must be explicit because this is a copied script, not merely a
    paraphrased reference.
