## Overview

Implement the first usable Universo-backed PlayCanvas Editor authoring flow: open the real Editor from a metahub, bootstrap a selected/default PlayCanvas project, handshake through a typed iframe bridge, save a simple scene through Platformo storage, close/reopen, and verify the persisted state with browser evidence.

This plan intentionally targets a bounded first slice. It does not implement PlayCanvas Cloud parity, S3/provider administration, binary asset pipelines, Colyseus authoring, AI/MCP automation, or direct runtime scene rendering. Editor save remains metahub authoring state only; publication/export/application sync stay explicit.

## Planning Decisions

-   Use a **hybrid seam** for the first implementation: real PlayCanvas Editor artifact + Universo bootstrap/config injection + typed bridge/storage adapter. Pure postMessage-only is not assumed sufficient because upstream Editor depends on `window.config`, REST globals, realtime scene/asset flows, and script-parse pipeline behavior.
-   Keep the first data path **scene-only plus minimal JSON metadata** until the real boot/handshake/save/reopen flow is proven. Script assets, script bindings, generated artifacts, publish/export, and runtime manifests remain capability-gated.
-   Keep the current **manager-only permission model**. Do not add read-only Editor mode in this slice unless implementation uncovers a hard product blocker and the permission/API contract is explicitly added.
-   Keep `sandbox="allow-scripts"` as the default target. Any `allow-same-origin`, CSP, worker, popup, or connect-src relaxation must be an explicit reviewed change with browser proof.
-   Use shared contracts in `@universo-react/types`; use package-local or domain-local helpers first, and touch `@universo-react/utils` only if a genuinely reusable executor/auth/error/schema helper is needed.
-   Use UUID v7 for any new ids generated by this slice.

## Architecture Contract

### Layer Ownership

-   Metahub owns PlayCanvas authoring projects, scenes, metadata, files, package attachment config, and Editor host settings.
-   The Editor iframe owns upstream Editor UI/runtime internals only while mounted.
-   The MUI host owns auth context, selected/default project, bridge session lifecycle, dirty-close guard, localized user states, and TanStack Query invalidation.
-   Published applications own only explicit publication/runtime manifest output, not live authoring files.

### Bridge Contract

Create shared bridge contracts in `@universo-react/types`, for example:

```ts
export const PLAYCANVAS_EDITOR_BRIDGE_VERSION = '1' as const

export const playCanvasBridgeCommandSchema = z.discriminatedUnion('type', [
    z.strictObject({
        type: z.literal('editor.ready'),
        requestId: uuidV7StringSchema,
        bridgeVersion: z.literal(PLAYCANVAS_EDITOR_BRIDGE_VERSION),
        sessionId: uuidV7StringSchema,
        nonce: z.string().min(32),
        capabilities: z.array(z.string()).max(32)
    }),
    z.strictObject({
        type: z.literal('scene.save'),
        requestId: uuidV7StringSchema,
        sessionId: uuidV7StringSchema,
        nonce: z.string().min(32),
        projectId: uuidV7StringSchema,
        sceneId: uuidV7StringSchema,
        expectedCurrentChecksum: z
            .string()
            .regex(/^[a-f0-9]{64}$/i)
            .nullable(),
        payload: boundedPlayCanvasScenePayloadSchema
    })
])

export type PlayCanvasBridgeCommand = z.infer<typeof playCanvasBridgeCommandSchema>
```

Validation rule: all incoming iframe and backend bridge payloads use `safeParse`. Raw Zod errors never reach normal UI; the host maps machine-readable codes to localized messages. `uuidV7StringSchema` should use an existing project UUID v7 helper if one exists; otherwise add a small shared validator and tests instead of accepting arbitrary UUID versions for newly generated bridge ids. Scene payload validation must be bounded: max serialized size aligned with the existing 5 MiB file limit, strict envelopes, finite depth/array limits, no unknown top-level fields unless explicitly versioned, and no prototype/function-bearing objects after JSON parse.

### Storage Adapter Contract

The adapter must preserve existing PlayCanvas project write ordering:

```ts
async function saveSceneThroughAdapter(input: SaveSceneInput): Promise<SaveSceneResult> {
    const metadata = await upsertSceneMetadata({
        projectId: input.projectId,
        sceneId: input.sceneId,
        payloadFile: input.payloadFile,
        expectedVersion: input.expectedSceneVersion
    })

    const file = await writeProjectFile({
        path: metadata.payloadFile.path,
        mime: 'application/json',
        contentBase64: encodeJson(input.payload),
        expectedCurrentChecksum: input.expectedCurrentChecksum
    })

    return refreshSceneAfterFileWrite({
        sceneId: metadata.id,
        checksum: file.checksum
    })
}
```

No free/orphan file writes. Metadata path first, guarded file write second, metadata/status refresh third.

First-slice scene save algorithm:

1. Resolve the attached PlayCanvas package and selected/default project from package config.
2. Resolve the selected/default scene by human-readable metadata; create scene metadata if it does not exist yet.
3. Build or verify the scene payload path through the existing PlayCanvas project file service helpers, not ad hoc string concatenation.
4. Upsert scene metadata with `payloadFile` in a non-publishable `pending`/`missing` status.
5. Write the JSON scene file with `expectedCurrentChecksum` and existing path/MIME/size guards.
6. Mark scene metadata ready with the new checksum/summary.
7. On failure after metadata upsert or file write, restore previous metadata or leave an explicit non-publishable missing status. Add orphan and stale-metadata tests for each failure point.

Required adapter/store contract:

-   Add or expose an adapter-level method that can create a missing scene metadata row with a valid `payloadFile`, capture the previous row/state for rollback, and map optimistic-version conflicts to bridge-safe conflict codes.
-   Existing low-level calls may be reused only when they preserve this contract. If they cannot create the missing-scene path or restore previous metadata safely, add narrow store/service methods behind the existing `DbExecutor` boundary.
-   Cover create-new scene, update-existing scene with expected version, concurrent stale checksum, file-write failure after metadata create, and mark-ready failure after physical file write.

## UI Contract

### Host Page

-   Surface: `PlayCanvasEditorHostPage` route, full-height MUI page using existing `Stack`, `Alert`, `Button`, toolbar/header patterns.
-   Controls:
    -   Back to packages button.
    -   Open separately button when configured.
    -   Save status indicator from bridge state.
    -   Dirty-close guard when the iframe reports unsaved changes.
    -   Optional project/scene selector only by human-readable names/codenames, never UUIDs.
-   Component reuse:
    -   New `EditorHostToolbar`, `EditorStatusAlert`, `EditorDirtyCloseDialog`, and selector components must be thin feature compositions, not new visual primitives.
    -   Dirty/conflict flows must use `StandardDialog`; list/empty states must use existing table/list/empty primitives where they fit.
-   Hidden/system fields:
    -   `metahubId`, `packageSlug`, `projectId`, `sceneId`, artifact token, bridge nonce, checksum, capability ids, file paths.
-   Display values:
    -   Project and scene names use localized display text.
    -   Technical compatibility states map to localized labels like "Editor unavailable", "Project needs migration", "Save conflict".
-   Validation:
    -   No raw Zod/internal/backend messages.
    -   User-facing EN/RU copy for loading, ready, permission denied, artifact missing, artifact stale, handshake failed, save failed, conflict, unsupported version, dirty unsaved changes.
-   Responsive proof:
    -   Desktop `1920x1080`, tablet `768x1024`, mobile `390x844`.
    -   No document-level horizontal overflow. Internal Editor scrolling is allowed only inside the iframe.

### Iframe Area

-   `sandbox="allow-scripts"` by default, `referrerPolicy="no-referrer"`, no raw artifact URL shown to the user.
-   Bridge must explicitly support opaque-origin mode:
    -   validate `event.source`;
    -   validate session id and nonce;
    -   validate strict message schema;
    -   use request ids and expiry;
    -   document targetOrigin tradeoff for host-to-frame messages.
-   Focus/keyboard:
    -   User can tab to host controls, enter iframe, and escape/shift-tab back to host controls without being trapped.
    -   Dirty close guard does not block browser indefinitely and has localized actions.

### Save And Conflict UI

-   Save success shows localized non-technical confirmation.
-   Stale checksum/multi-tab conflict shows localized recovery choices:
    -   reload latest scene;
    -   keep local changes and retry after reload if supported;
    -   cancel.
-   Never show raw checksum, file path, UUID-only record, JSON payload, stack trace, or bridge envelope.

### Project/Scene Lists

-   If added, reuse existing table/list/dialog primitives.
-   Reuse existing `@universo-react/template-mui` primitives before adding UI components:
    -   `FlowListTable` for project/scene list surfaces;
    -   `StandardDialog` for conflict and dirty-close flows;
    -   `ConfirmDeleteDialog` only if a destructive scene action is explicitly added later;
    -   `EmptyListState` for missing project/scene/artifact states.
-   Keep the existing `withMetahubDialogSettings` integration intact for metahub-owned settings dialogs.
-   Description/notes fields are multiline by default.
-   Structured metadata renders as labels/badges/previews or remains hidden; no raw JSON cells.

## User-Facing Leakage Ban

Normal UI must not expose:

-   `metahubId`, `packageSlug`, `projectId`, `sceneId`, user/session ids, artifact tokens, bridge nonce/session ids.
-   File paths such as `playcanvas-projects/...`, checksums, raw capability names, raw bridge command names.
-   Raw JSON/object cells, `[object Object]`, Zod errors, backend stack traces, `ERR_*` strings, protocol envelopes, unlocalized English fallback in RU views.

Tests must include explicit anti-leakage assertions.

-   [ ] First slice session strategy: use short-lived HMAC-signed bridge session payloads plus a server-side replay/idempotency store.     -   Do not put session ids, HMAC tokens, nonces, artifact tokens, checksums, or file paths in route params or query strings.
-   [ ] Route placement decision:
    -   keep host descriptor and tokenized artifact routes in `domains/packages`;

## Potential Challenges

-   Real upstream Editor may require more REST/realtime behavior than the first slice should implement. Mitigation: add a boot proof gate and document missing upstream contracts before expanding scope.
-   Opaque-origin iframe messaging complicates `targetOrigin`. Mitigation: keep source/session/nonce/schema validation mandatory and document any target-origin compromise.
-   Tokenized nested asset URLs can fail for lazy chunks/workers after the HTML loads. Mitigation: relative base strategy, manifest checks, Playwright network assertions.
-   Scene payload shape may not match upstream Editor internals. Mitigation: define a minimal accepted scene shape and reject/disable unsupported operations with localized messages.
-   Node `>=22.22.0` for Editor artifact build may not match all developer/CI environments. Mitigation: package-local build remains separate; root build enrollment is not assumed.
-   UI can easily become technical. Mitigation: enforce the leakage ban and browser assertions before accepting implementation.
