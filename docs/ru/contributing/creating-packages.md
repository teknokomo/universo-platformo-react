## Обзор

Новые пакеты создаются по стандартной структуре в директории `packages/`. Фронтенд-пакеты используют TypeScript (TSX) с двойной сборкой CJS/ESM. Бэкенд-пакеты используют TypeORM для доступа к базе данных.

## Структура пакета

```
packages/your-package/base/
├── src/
│   └── index.ts
├── package.json
├── tsconfig.json
└── README.md
```

## Ключевые правила

* Фронтенд-пакеты должны компилироваться в CommonJS и ES Modules
* Бэкенд-пакеты должны использовать паттерн TypeORM Repository
* Все пакеты должны регистрировать сущности и миграции централизованно
* Для кросс-пакетных импортов используйте имена пакетов pnpm workspace
