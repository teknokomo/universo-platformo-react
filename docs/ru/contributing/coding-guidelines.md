# Стандарты кода

## Общие правила

- Предпочитайте TypeScript там, где пакет уже использует его.
- Используйте отступ 2 пробела и держите изменения сфокусированными.
- Держите корневую и пакетную документацию согласованной с реальным кодом.
- Не возвращайте в новый код удалённые TypeORM-first паттерны.

## Правила по пакетам

- Используйте workspace package names для межпакетных импортов.
- Backend-доступ к базе должен идти через SQL-first stores или executor-helper.
- Фронтенд-пакеты должны вписываться в текущую React-оболочку и общие UI-паттерны.
- Обновляйте English и Russian docs вместе, когда существует переводная пара.

Пишите изменения так, чтобы они соответствовали модульному и repository-first вектору платформы.
