# OntoIndex: интеллект по коду

OntoIndex строит локальный граф этого репозитория и открывает его разработчикам
и AI-агентам через MCP-сервер и CLI. Он помогает отвечать на вопросы уровня
графа до правок: где используется символ, какова зона влияния изменения,
а также контекст вызовов и маршрутов вокруг него.

## Что это (и что нет)

OntoIndex индексирует исходный код и Markdown в локальный граф. Это инструмент
разработчика и агента, а не функция продукта.

-   Это интеллект по исходным файлам, а не движок запросов OWL, RDF или SPARQL.
-   Это не поиск по данным во время работы, таким как записи метахаба или строки пользователей.
-   Пакет `ontograph/ontoindex` не связан с похоже названными крейтами Rust.

Никогда не ставьте похоже названный пакет, ожидая этот инструмент.

## Требования

-   Node.js `>=22.6.0` (уже требуется этим репозиторием).
-   Утилита командной строки `git`.
-   В Linux — `python3`, `make` и компилятор C++ для сборки нативных парсеров.

## Установка

Ставьте закреплённый релиз для каждого разработчика. Это внешний инструмент,
поэтому он не добавляется в зависимости репозитория.

```bash
npm install -g https://github.com/ontograph/ontoindex/releases/download/v1.9.10/ontoindex-1.9.10.tgz
ontoindex --version
```

## Первая индексация

Постройте граф из корня репозитория. Скрипт-обёртка передаёт команды
установленному инструменту и печатает подсказку, если он отсутствует.

```bash
pnpm ontoindex:analyze
pnpm ontoindex:status
```

Граф хранится в `.ontoindex/` и игнорируется git. Глобальный реестр
живёт в `~/.ontoindex/` и хранит только пути и метаданные репозиториев.

## Настройка MCP для Claude Code

Репозиторий поставляет проектный `.mcp.json`, который объявляет сервер. Claude
Code одобряет проектные серверы автоматически через существующую настройку,
поэтому дополнительный шаг не нужен.

```json
{
    "mcpServers": {
        "ontoindex": {
            "command": "ontoindex",
            "args": ["mcp"]
        }
    }
}
```

Проверьте подключение командой `claude mcp list`.

## Настройка MCP для других агентов

Codex, Gemini и Qoder используют свои форматы настройки для каждого разработчика
и не коммитятся в репозиторий. Добавьте сервер документированной командой.

```bash
codex mcp add ontoindex -- ontoindex mcp
```

## Повседневное использование

Запрашивайте граф через CLI или через инструменты MCP в сессии агента.

```bash
pnpm ontoindex:status
node tools/ontoindex/run-ontoindex.mjs impact validateUser --depth 2
pnpm ontoindex:changes
```

После получения значительных изменений пересоберите граф, чтобы ответы не
устаревали. Используйте `pnpm ontoindex:analyze` для полного прохода или
`pnpm ontoindex:changes` для частичного обновления.

## Ограничения

OntoIndex поддерживает TypeScript и JavaScript как входы первого класса. Этот
репозиторий ориентирован на SQL, поэтому качество связей внутри сырого SQL и DDL
в `supabase/` стоит проверять в ходе работы. Считайте инструмент сильной помощью
для вопросов по TypeScript, маршрутам и графу вызовов, а для слоя SQL сохраняйте
прямой просмотр.

## Устранение неполадок

-   Если обёртка сообщает, что инструмента нет, переустановите закреплённый релиз.
-   Если результаты не отражают свежий код, снова запустите `pnpm ontoindex:analyze`.
-   Каталог `.ontoindex/` только локальный и никогда не должен коммититься.
