# Модель данных трактовочной сети

Модель данных трактовочной сети кодирует универсальную трактовочную сеть: структуры, ячейки матрицы, необязательные строки режима совместимости и материалы, прикреплённые к ячейкам.

## Модель первого этапа

Этап 1 поставляет runtime-модель прямо в product fixture и в workspace-виджете:

-   `Structure` отображается пользователю как Структура и использует поля «Название» и необязательное «Описание» в форме создания/редактирования.
-   `Interpretation` хранит матрицу конкретной структуры.
-   `Relation` остаётся доступной моделью для типизированных связей между структурами, трактовками и ячейками.
-   `Material` использует Page-like поля и хранит Editor.js-контент, привязанный к ячейкам.
-   Перечисления `Context`, `RelationType` и `CellColor` управляют отображением и подписями связей.
-   Workspace-записи `TableTemplate` позволяют пользователям создавать или копировать reusable matrix structures внутри опубликованного приложения.

## Атрибуты ячеек

Каждая ячейка Interpretation Matrix использует плоские поля вместо вложенного JSON:

-   `RowLabel` и `ColLabel` — задаваемые пользователем локализованные подписи строки и колонки.
-   `CellValue` — задаваемое пользователем локализованное название ячейки.
-   `CellDescription` — необязательное задаваемое пользователем локализованное многострочное описание.
-   `CellId` — системный UUID v7.
-   `ParentCellId` — скрытый системный необязательный родитель по `CellId` для иерархической матрицы.
-   `RowKey` и `ColKey` — скрытые системные стабильные ключи осей.
-   `CellFillColor` — ссылка на `CellColor`.
-   `Border{Top,Right,Bottom,Left}{Color,Width,Style}` — плоские controls границ.
-   `MaterialRef` — необязательная ссылка на страницу Material.
-   `_tp_sort_order` — скрытый системный сохраняемый порядок среди соседних ячеек. `Depth` вычисляется во время выполнения и не хранится.

В диалоге ячейки пользователь задаёт подписи строки и колонки, название ячейки и необязательное многострочное описание. При создании ячейки в существующей строке её подпись может наследоваться; при создании строки в существующей колонке может наследоваться подпись колонки. Пользователь может изменить унаследованное значение до сохранения. Тексты-заполнители «Новая строка» и «Новая ячейка» не сохраняются как пользовательские данные.

По умолчанию конфигурация виджета `interpretationNetworkWorkspace` использует `matrixMode: "hierarchicalCells"`. В этом режиме первая ячейка новой структуры называется `Universe` / `Вселенная`, а следующие ячейки могут участвовать в иерархии родителей и потомков. Режим `independentRows` сохраняет независимое поведение строк для конфигураций, где оно выбрано явно.

## Настройки отображения Матрицы

Настройки отображения Матрицы являются конфигурацией виджета, а не данными ячеек:

-   `matrixMode` управляет семантикой данных Матрицы: `hierarchicalCells` или `independentRows`.
-   `allowedMatrixViews` задаёт непустой набор равноправных представлений Матрицы: `table`, `horizontalRows` и `verticalTree`.
-   `defaultMatrixView` входит в `allowedMatrixViews` и открывается при входе в рабочее пространство.
-   `tableProjection` управляет табличным представлением. Значение шаблона по умолчанию — `hierarchicalPath`: родительские ячейки становятся хлебными крошками, а дочерние ячейки текущего уровня становятся строками. `independentAxes` остаётся доступным для явной табличной проекции со строками и колонками.
-   `breadcrumbDepth` по умолчанию показывает полный путь. Ограничение `last` показывает заданное число последних уровней и открывает скрытых родителей через локализованное меню с многоточием.
-   `toolbarLayout` по умолчанию равен `horizontal`; `vertical` доступен как явная настройка для плотных рабочих пространств.
-   `showHierarchicalTableHeaders` по умолчанию равен `false` для иерархической Таблицы. Его можно включить, чтобы показывать заголовки колонок текущего уровня и ячейки.
-   `showHierarchicalTableHeaderCard` по умолчанию равен `true`, чтобы выбранная родительская ячейка оставалась отдельной карточкой контекста над таблицей строк до перехода в хлебные крошки.
-   `colorBreadcrumbsByCell` по умолчанию равен `true`, поэтому боксы хлебных крошек используют тот же цвет заливки, что и исходные ячейки, и отдельный эффект наведения и фокуса для навигационной обратной связи.

Единая схема и нормализаторы находятся в `@universo-react/types` (`common/applicationLayouts`). Представления Матрицы являются разными отображениями одной Матрицы, а не отдельными функциями «Матрица» и «Таблица». Требуется хотя бы одно представление, повторы не допускаются, а представление по умолчанию должно входить в разрешённый набор. `verticalTree` требует `hierarchicalCells`; `table` и `horizontalRows` также поддерживают `independentRows`. Шаблон задаёт канонические настройки на уровне метахаба; Панель приложения может переопределить их для конкретного развёртывания; пользователи рабочего пространства переключаются между разрешёнными представлениями при работе с содержимым.

Эти поля не требуют миграции базы данных, нового пресета сущности или увеличения версии шаблона метахаба.

## UI contract

Runtime-виджет показывает список структур слева. После открытия структуры матрица отображается во вкладке «Матрица», чтобы следующие этапы могли добавить другие вкладки уровня структуры без изменения route-контракта. Правая панель зарезервирована только для материалов выбранной ячейки матрицы. Пользователи видят названия, описания и Editor.js-содержимое материалов; raw UUID, raw JSON и внутренние имена полей на обычных экранах не показываются.

Настройки приложения не выводят поведение трактовочной сети из захардкоженных LMS-настроек. Страница настроек читает активные материализованные виджеты макета; настройки матрицы сохраняются обратно в конфигурацию виджета `interpretationNetworkWorkspace`, а «Учебный контент» показывается только для приложений, runtime-состояние которых действительно содержит такую конфигурацию.

Представления `table`, `horizontalRows` и `verticalTree` показывают одни и те же записи ячеек. По умолчанию Таблица строится от иерархии: путь выбранной ячейки становится кликабельными хлебными крошками с цветом ячейки, выбранный родитель становится текущим контекстом таблицы, прямые дочерние ячейки становятся строками, а заголовки колонок скрыты до явного включения. Нажатие на хлебные крошки обновляет выбор, route state, видимых потомков и панель Материалов. Когда `tableProjection` равен `independentAxes`, Таблица использует пользовательские `RowLabel` и `ColLabel` как семантические заголовки; пустое пересечение `(RowKey, ColKey)` остаётся пустой локализованной ячейкой таблицы и не сохраняется как новая запись. Пользовательские экраны скрывают UUID, ключи осей, родительские идентификаторы, порядок сортировки, сырой JSON и старые продуктовые названия.

## Workspace templates

Workspace-scoped table template можно создать или скопировать внутри опубликованного приложения и затем использовать для создания видимой матрицы. Это часть Stage-1 product contract.

## Ограничения

-   В этапе 1 нет IPFS.
-   Нет увеличения версии схемы или шаблона.
-   Базовые примитивы шаблона остаются основой; workspace widget композит их вместо возврата demo dashboard surfaces.
-   Проверка фикстур и E2E должна использовать сгенерированные снапшоты, минимальную локальную Supabase, контрактные проверки и браузерные подтверждения для всех трёх адаптивных представлений Матрицы на русском и английском языках.
