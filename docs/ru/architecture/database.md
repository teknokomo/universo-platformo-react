# Проектирование базы данных

Текущий репозиторий использует PostgreSQL, обычно через Supabase, с общим
рантаймом на базе Knex.

## Текущая модель

- `@universo-react/database` владеет общим рантаймом Knex и executor-helper.
- Выполнение, привязанное к запросу, используется там, где нужен auth- и RLS-контекст.
- `@universo-react/migrations-platform` регистрирует и запускает платформенные миграции.
- `@universo-react/schema-ddl` даёт генерацию схем во время выполнения и утилиты diff.

## Цели проектирования

Слой базы данных строится ради контролируемой эволюции схем, soft-delete-aware
бизнес-доменов, прозрачности миграций и экспортируемых определений.

Это делает слой данных частью ядра платформы и её рабочей модели.
