# Архитектура бэкенда

Бэкенд строится вокруг `@universo/core-backend`, который поднимает Express
приложение, инициализирует общий рантайм Knex, выполняет платформенные миграции,
монтирует сервисные маршруты и раздаёт фронтенд-бандл.

## Паттерн рантайма

- Express обеспечивает HTTP-маршрутизацию и композицию middleware.
- Функциональные пакеты владеют маршрутами, сервисами и SQL-first persistence helper.
- `@universo/migrations-platform` валидирует и запускает платформенные миграции.
- `@universo/database` даёт общий singleton Knex и фабрики executor.

## Текущая доменная поверхность

Активная публичная поверхность бэкенда включает auth, onboarding, profiles,
metahubs, publications, applications, admin-потоки и поддержку OpenAPI-документации.

Это модульный бизнес-бэкенд для общих сервисов платформы.
