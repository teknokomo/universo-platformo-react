# Аутентификация и авторизация

Текущая auth-модель сочетает обработку сессий Passport.js с Supabase identity
и контекстом базы данных, привязанным к запросу.

## Текущая модель

- Бэкенд-маршруты auth и middleware живут в `@universo/auth-backend`.
- Фронтенд-хелперы auth и UI-потоки живут в `@universo/auth-frontend`.
- Клиенты используют запросы с учётом сессии и CSRF-защиту.
- Для защищённых операций с базой применяется RLS-контекст, привязанный к запросу.

## Важные контракты

- Публичные маршруты задаются централизованно в общих auth-утилитах.
- CSRF-сбои мапятся в HTTP 419 и безопасно ретраятся только один раз.
- Обработка 401 координируется с фронтенд-редиректами.
- Проверки членства и ролей остаются явной заботой платформы.

Эта модель поддерживает многопользовательские бизнес-потоки в общих модулях платформы.
