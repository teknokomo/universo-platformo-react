# Аутентификация

Большинство непубличных API-маршрутов требует аутентифицированной сессии.

## Текущий контракт

- Обработка сессий на бэкенде реализована через Passport.js.
- В качестве базовой системы аутентификации используется Supabase identity.
- Для изменяющих состояние запросов ожидается CSRF-защита.
- Ответы 401 и 419 имеют специальное поведение на фронтенде.

## Практическая рекомендация

Если вы интегрируетесь с запущенным экземпляром, исходите из того, что
защищённым маршрутам нужны и сессионная аутентификация, и корректная
CSRF-обработка, а не только поток с Bearer-токеном.
