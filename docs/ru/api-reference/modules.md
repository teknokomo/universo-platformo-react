# API модулей

Modules API охватывает маршруты проектирования в метахабе и маршруты доставки рантайма в приложении.
Он удерживает библиотеки рабочего пространства ресурсов, потребляющие модули, клиентские бандлы и серверные вызовы в рамках одного закрытого контракта.

## Конечные точки проектного метахаба

-   `GET /metahub/{metahubId}/modules` перечисляет проектные модули для одной области привязки.
-   `POST /metahub/{metahubId}/modules` создаёт проектный модуль.
-   `GET /metahub/{metahubId}/module/{moduleId}`, `PATCH /metahub/{metahubId}/module/{moduleId}` и `DELETE /metahub/{metahubId}/module/{moduleId}` читают, обновляют или удаляют один модуль.
-   Проектирование в рабочем пространстве ресурсов должно всегда сочетать `attachedToKind=general` с `moduleRole=library`.

Payload создания/обновления поддерживает:

-   `storageMode: "inline"` с `sourceCode`, стандартный режим хранения в базе;
-   `storageMode: "file"` с `sourcePath` и необязательным `sourceCode`; если `sourceCode` передан, бэкенд атомарно записывает его в настроенный source root, иначе читает существующий файл.

`sourcePath` — относительный путь `modules/**/*.ts` или `modules/**/*.tsx`. Абсолютные пути, parent traversal, скрытые сегменты, выход через symlink и JavaScript/ESM-расширения отклоняются.
Проектные ответы list/read могут включать метаданные `sourceStorage`. Вкладка «Модули», доступная только менеджерам метахаба, может показывать фактический путь исходного файла и checksum, чтобы автор мог сверить внешние изменения; опубликованные runtime-endpoint-ы эти authoring-only поля не раскрывают.

## Конечные точки рантайма приложения

-   `GET /applications/{applicationId}/runtime/modules` перечисляет метаданные опубликованных рантайм-модулей без встроенных бандлов.
-   `GET /applications/{applicationId}/runtime/modules/{moduleId}/client` возвращает кэшируемый клиентский JavaScript-бандл.
-   `POST /applications/{applicationId}/runtime/modules/{moduleId}/call` выполняет только разрешённые серверные методы без lifecycle.
-   Рантайм-маршруты раскрывают потребляющие модули, а не прямые строки библиотек из рабочего пространства ресурсов.

## Примечания fail-closed

-   Неподдерживаемые imports, версии SDK или сочетания scope-role отклоняются до рантайм-доставки.
-   Ошибки удаления, переименования и циклической зависимости общей библиотеки блокируют публикацию и последующую синхронизацию рантайма.
-   Публичный рантайм RPC требует `rpc.client` и не может вызывать lifecycle handlers.
-   Синхронизация приложения падает, а не молча пропускает сохранение рантайм-модулей.
-   Существующие схемы метахабов без колонок файловых источников не поддерживают file-backed команды; перед включением файлового хранения создайте свежую схему или выполните отдельный миграционный план.
