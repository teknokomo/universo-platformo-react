# API скриптов

Scripts API охватывает маршруты проектирования в метахабе и маршруты доставки рантайма в приложении.
Он удерживает библиотеки рабочего пространства ресурсов, потребляющие скрипты, клиентские бандлы и серверные вызовы в рамках одного закрытого контракта.

## Конечные точки проектного метахаба

- `GET /metahub/{metahubId}/scripts` перечисляет проектные скрипты для одной области привязки.
- `POST /metahub/{metahubId}/scripts` создаёт проектный скрипт.
- `GET /metahub/{metahubId}/script/{scriptId}`, `PATCH /metahub/{metahubId}/script/{scriptId}` и `DELETE /metahub/{metahubId}/script/{scriptId}` читают, обновляют или удаляют один скрипт.
- Проектирование в рабочем пространстве ресурсов должно всегда сочетать `attachedToKind=general` с `moduleRole=library`.

## Конечные точки рантайма приложения

- `GET /applications/{applicationId}/runtime/scripts` перечисляет метаданные опубликованных рантайм-скриптов без встроенных бандлов.
- `GET /applications/{applicationId}/runtime/scripts/{scriptId}/client` возвращает кэшируемый клиентский JavaScript-бандл.
- `POST /applications/{applicationId}/runtime/scripts/{scriptId}/call` выполняет только разрешённые серверные методы без lifecycle.
- Рантайм-маршруты раскрывают потребляющие скрипты, а не прямые строки библиотек из рабочего пространства ресурсов.

## Примечания fail-closed

- Неподдерживаемые imports, версии SDK или сочетания scope-role отклоняются до рантайм-доставки.
- Ошибки удаления, переименования и циклической зависимости общей библиотеки блокируют публикацию и последующую синхронизацию рантайма.
- Публичный рантайм RPC требует `rpc.client` и не может вызывать lifecycle handlers.
- Синхронизация приложения падает, а не молча пропускает сохранение рантайм-скриптов.
