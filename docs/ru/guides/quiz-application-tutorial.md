# Руководство по приложению-квизу

Это руководство повторяет реальный сценарий в браузере, который используется в Playwright-покрытии репозитория.
В конце у вас будет скрипт виджета квиза в metahub, макет с этим виджетом, связанное приложение и проверенное выполнение квиза.

## Перед началом

- Используйте окружение, где metahubs, publications и applications уже доступны.
- Войдите пользователем, который может управлять целевым metahub и связанным приложением.
- Учитывайте текущий контракт scripting: поддерживаются только встроенные скрипты и `sdkApiVersion = 1.0.0`.

## Шаг 1: Откройте или создайте metahub

1. Откройте целевой metahub из основного списка.
2. Убедитесь, что у вас есть доступ к General и Publications, а затем используйте вкладку Layouts внутри General.
3. Если вы стартуете с нуля, сначала создайте metahub и оставьте его default branch активной.

## Шаг 2: Создайте скрипт виджета квиза

1. Откройте диалог редактирования metahub и переключитесь на вкладку Scripts.
2. Создайте новый скрипт с ролью `widget`.
3. Используйте `quiz-widget` как codename.
4. Оставьте тип источника значением Embedded.
5. Сохраните скрипт после вставки исходника виджета квиза или после адаптации стартового шаблона.

![Вкладка скриптов metahub](../assets/quiz-tutorial/metahub-scripts.png)

Сценарий Playwright в браузере использует этот шаг, чтобы подтвердить, что скрипт квиза виден в реальной области Scripts на уровне metahub.

## Шаг 3: Подключите виджет к макету

1. Откройте Common -> Layouts и выберите макет, который должен содержать квиз.
2. Добавьте или отредактируйте виджет `quizWidget` в center zone.
3. Установите `scriptCodename` в `quiz-widget`.
4. Сохраните макет и убедитесь, что виджет остаётся видимым на странице деталей макета.

![Макет с виджетом квиза](../assets/quiz-tutorial/layout-quiz-widget.png)

В текущем контракте квиза виджет расположен по центру, чтобы runtime был сосредоточен на ходе вопросов, а не на боковых панелях.

## Шаг 4: Опубликуйте metahub и создайте приложение

1. Создайте или обновите publication из подготовленного состояния metahub.
2. Дождитесь, пока publication получит готовую активную версию.
3. Создайте связанное приложение из этой publication или обновите уже существующее связанное приложение.
4. Запустите создание схемы или синхронизацию, если приложению всё ещё нужны runtime-таблицы.

Именно в этой точке данные publication, layout и бандлы скриптов переходят со стороны metahub на сторону application.

## Шаг 5: Настройте параметры приложения

1. Откройте панель управления приложением по адресу `/a/:applicationId/admin`.
2. Перейдите в Settings.
3. Используйте вкладку General, чтобы выбрать предпочитаемый размер окна, поведение fullscreen, возможность изменения размера и способ закрытия.
4. Сохраните параметры и подтвердите, что следующие диалоги панели управления используют тот же presentation contract.

![Вкладка общих параметров приложения](../assets/quiz-tutorial/application-settings-general.png)

Эти параметры влияют только на панель управления приложением.
Они не меняют содержимое квиза или макет, которыми владеет metahub publication.

## Шаг 6: Проверьте runtime квиза

1. Откройте публичный runtime по адресу `/a/:applicationId`.
2. Дождитесь, пока виджет загрузит первый вопрос.
3. Отправьте ответ и убедитесь, что виджет переходит к следующему шагу сценария.
4. Завершите квиз и проверьте итоговую сводку результатов.

![Публичное выполнение виджета квиза](../assets/quiz-tutorial/runtime-quiz.png)

Текущий стартовый шаблон ожидает клиентский метод `mount()` и серверный метод `submit()`, поэтому рабочий runtime одновременно подтверждает и клиентский, и серверный путь выполнения скрипта.

## Диагностика проблем

- Если скрипт сохраняется, но не появляется в runtime, заново опубликуйте metahub и пересинхронизируйте application.
- Если виджет рендерится без данных, проверьте, что `scriptCodename` виджета в layout точно совпадает с сохранённым codename скрипта.
- Если вызовы runtime RPC падают, проверьте, что роль скрипта и capability по-прежнему открывают клиентский RPC-путь.
- Если диалоги панели управления игнорируют предпочитаемый размер, проверьте, что вы меняли именно параметры application, а не metahub или admin settings.
- Если нужен эталонный пример, посмотрите `tools/testing/e2e/specs/generators/metahubs-quiz-app-export.spec.ts` и сохранённый fixture `tools/fixtures/metahubs-quiz-app-snapshot.json`.
