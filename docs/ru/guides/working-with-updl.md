# Работа с UPDL

К UPDL стоит подходить как к слою структурированного моделирования переносимых
определений платформы и общих структур платформы.

## Рекомендуемый подход

1. Начинайте с домена, который хотите описать, а не с визуальных эффектов.
2. Выбирайте подходящие семейства узлов для контекста, объектов, поведения, событий и данных.
3. Держите модель достаточно явной, чтобы потом её валидировать, версионировать и экспортировать.
4. Связывайте UPDL-работу с publications, applications и другими структурами платформы.
5. Ставьте переиспользование и ясность выше, чем размер графа.

Такой подход удерживает UPDL в согласии с более широким направлением платформы.
