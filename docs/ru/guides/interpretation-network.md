# Трактовочная сеть

Шаблон **Трактовочная сеть** добавляет в каталог метахабов Universo Platformo реальное рабочее пространство трактовочной сети. Это поставка первого этапа для универсальных приложений трактовочной сети, и она включает:

-   Предзаданную runtime-модель **Concept / Interpretation / Relation / Material** поверх базовых пресетов `object`, `page` и `enumeration`.
-   Три закрытых перечисления — **Context**, **RelationType** и **CellColor** — которые управляют стилем ячеек и связями графа.
-   Переиспользуемый runtime-виджет **interpretationNetworkWorkspace** в `packages/universo-react-apps-template-mui`, который рендерит двухпанельное рабочее пространство без мусорного dashboard-слоя.
-   Отдельный виджет поля `cellStylePicker` для управления цветом и границами ячеек матрицы.
-   Playwright-генератор продукта, который выпускает `tools/fixtures/metahubs-interpretation-network-app-snapshot.json`, и строгий контракт фикстуры, который отклоняет лишние dashboard-виджеты.

## Когда использовать

-   Вам нужен мульти-контекстный граф знаний: термины с несколькими трактовками в зависимости от домена.
-   Вам нужен стиль ячеек (фоновый цвет + границы по сторонам), который round-trip-ится через конверт снапшота.
-   Вам нужны типизированные рёбра между понятиями, трактовками и отдельными ячейками.
-   Вам нужен workspace-scoped поток шаблонов таблиц, чтобы пользователи могли создавать или копировать reusable matrix structures внутри опубликованного приложения.

## Архитектура

Опубликованное приложение сохраняет обычную левую навигацию приложения. Центральное рабочее пространство использует новый виджет трактовочной сети и показывает:

-   левая панель: дерево структуры + матрица;
-   правая панель: материалы, привязанные к выбранной ячейке матрицы.

Рабочее пространство намеренно операционное, а не декоративное. Оно переиспользует общие runtime-primitive и общие CRUD/dialog/material controls вместо отдельного dashboard shell.

## Установка

```bash
pnpm supabase:e2e:start:minimal
pnpm env:e2e:local-supabase
pnpm run build:e2e:local-supabase
pnpm run test:e2e:interpretation-network-fixture-gate
pnpm run check:interpretation-network-fixture-contract
```

## Обзор конфигурации

Фикстура не создаёт предзаданные пользовательские структуры, ячейки матрицы или материалы. Она поставляет универсальную объектную модель, стартовую страницу, конфигурацию runtime-виджета и перечисления стилей, чтобы пользователь создавал собственные структуры уже внутри опубликованного приложения.

## Обзор двухпанельного рабочего пространства

Опубликованное приложение рендерит:

-   левая навигация приложения: обычный workspace/navigation shell;
-   трактовочное рабочее пространство: новый виджет `interpretationNetworkWorkspace`;
-   левая панель до открытия структуры: список структур и действие создания структуры;
-   левая панель после открытия структуры: матрица и действия внутри выбранной структуры;
-   правая панель: материалы, привязанные к выбранной ячейке матрицы.

## Стиль ячеек

Каждая ячейка матрицы несёт стабильный `CellId` (UUID v7) и плоские поля цвета и границ. Виджет `cellStylePicker` рендерит сетку чипов для выбора цвета, controls для ширины/стиля границ и живой preview.

## Прикрепление материалов

Материалы используют контракт, похожий на страницы: название, описание и блочное содержимое Editor.js. В интерфейсе приложения они называются материалами, привязаны к выбранной ячейке матрицы и поддерживают таблицу/карточки и фильтр по названию.

## Workspace table templates

Этап 1 включает workspace-scoped поток шаблонов таблиц. Пользователи могут создавать или копировать reusable matrix template внутри workspace и создавать видимые строки матрицы без metahub-authoring доступа.

## Ограничения

-   В этапе 1 нет IPFS/IPNS/content-addressed publication.
-   Нет увеличения версии схемы или шаблона метахаба.
-   Существующих workspace-ролевых моделей платформы достаточно для этапа 1.
