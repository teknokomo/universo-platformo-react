# Макеты приложения

Макеты приложения - это runtime-копии макетов, опубликованных из metahub, а также макеты, созданные прямо в приложении.
Они управляются в панели приложения: Приложения -> Макеты.
Панель приложения теперь переиспользует ту же shared authoring surface, что и детали макета в metahub, поэтому зоны виджетов, действия строк и карточно-списковое поведение остаются единообразными между design-time и application-side настройкой.

## Источники

У каждого макета есть источник:

- Metahub: создан при синхронизации схемы коннектора из снимка публикации.
- Приложение: создан или скопирован внутри панели управления приложением.

Источник показывается на карточках и в списковых представлениях, чтобы администратор отличал унаследованную конфигурацию от локальной.

## Локальные изменения

Администратор может:

- создавать общие и scoped к сущности макеты приложения;
- копировать metahub-макет в отдельный макет приложения;
- делать макет основным для его области;
- включать и выключать макеты;
- выключать, добавлять и настраивать виджеты макета.

Удаление metahub-макета в приложении не выполняет hard delete. Приложение помечает его как исключённый, выключенный и не основной, чтобы следующая синхронизация не восстановила его молча.
Макеты, созданные в приложении, удаляются через стандартный application soft-delete путь.

## Синхронизация

Когда синхронизация коннектора импортирует новую публикацию metahub:

- чистые metahub-макеты перезаписываются новым содержимым источника;
- локально изменённые metahub-макеты помечаются как конфликт и не перезаписываются;
- metahub-макеты, удалённые из публикации, помечаются как source removed;
- макеты приложения сохраняются;
- default из источника не заменяет default, созданный в приложении или локально изменённый в той же области.

Модель конфликтов по умолчанию работает fail-closed: она сохраняет локальную работу и показывает sync state для последующего решения.
Диалог diff коннектора также позволяет администратору явно выбрать перезапись, если при этой синхронизации содержимое макетов из metahub должно заменить локальные настройки metahub-derived макетов.

## Runtime-поведение

Runtime приложения использует только активные макеты и активные виджеты.
Неактивные макеты остаются доступны в панели управления и могут быть включены позже.
