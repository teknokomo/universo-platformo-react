# Экспорт и импорт снимков

Экспорт/импорт снимков позволяет сохранить полное состояние метахаба
(или конкретной версии публикации) в виде портативного JSON-конверта и восстановить
его в том же или другом экземпляре Universo Platformo.

## Обзор

**Конверт снимка** — это самодостаточный JSON-файл, который включает:

- **kind**: `metahub_snapshot_bundle`
- **bundleVersion**: `1` (для прямой совместимости)
- **snapshot**: полное состояние метахаба — entities, layouts, scripts, shared attributes/constants/values и shared override rows
- **snapshotHash**: хеш целостности SHA-256, вычисленный из нормализованных данных снимка

## Экспорт

### Из версии публикации

1. Перейдите в метахаб → Публикации → выберите публикацию.
2. В списке версий откройте меню действий для нужной версии.
3. Нажмите **Экспорт**.
4. Браузер загрузит файл `.json`, содержащий конверт снимка.

### Из метахаба напрямую

Используйте конечную точку API `GET /api/v1/metahub/:id/export` для получения
полного снимка метахаба без использования версий публикаций.

## Импорт

### Как новый метахаб

1. Перейдите на страницу списка метахабов.
2. Нажмите стрелку раскрытия на кнопке **Создать**.
3. Выберите **Импорт из снимка**.
4. Выберите файл конверта `.json` из файловой системы.
5. Система проверит конверт, верифицирует хеш целостности и создаст
   новый метахаб с восстановленными сущностями.

### Как версия публикации

1. Перейдите в метахаб → Публикации → выберите публикацию.
2. Нажмите стрелку раскрытия на кнопке **Создать версию**.
3. Выберите **Импорт из снимка**.
4. Выберите файл конверта `.json`.
5. Будет создана новая версия публикации из импортированных данных снимка.

## Спецификация формата файла

```json
{
  "kind": "metahub_snapshot_bundle",
  "bundleVersion": 1,
  "exportedAt": "2026-04-03T12:00:00.000Z",
  "snapshot": {
    "versionEnvelope": {
      "structureVersion": "0.1.0",
      "templateVersion": null,
      "snapshotFormatVersion": 2
    },
    "entities": { ... },
    "sharedAttributes": [ ... ],
    "sharedConstants": [ ... ],
    "sharedEnumerationValues": [ ... ],
    "sharedEntityOverrides": [ ... ]
  },
  "snapshotHash": "sha256-hex-string"
}
```

### Поля конверта

| Поле | Тип | Описание |
|------|-----|----------|
| `kind` | `string` | Всегда `metahub_snapshot_bundle` |
| `bundleVersion` | `number` | Версия схемы (текущая `1`) |
| `exportedAt` | `string` | Временная метка экспорта в формате ISO 8601 |
| `snapshot` | `object` | Полные данные состояния метахаба |
| `snapshotHash` | `string` | SHA-256 хеш нормализованного снимка |

## Shared Authoring Data

- `sharedAttributes`, `sharedConstants` и `sharedEnumerationValues` сохраняют общие пулы Resources workspace.
- `sharedEntityOverrides` сохраняет per-target exclusions, inactive overrides и sparse sort-order changes.
- Загрузка runtime publication materialize-ит эти shared sections до application sync и runtime deserialization.
- Snapshot import сначала пересоздаёт shared containers, а затем remap-ит цели shared overrides на восстановленные objects.

## Аспекты безопасности

- **Проверка хеша**: При импорте система пересчитывает SHA-256 хеш
  снимка и сравнивает его с `snapshotHash`. Изменённые файлы отклоняются
  с ошибкой `400 Bad Request`.
- **Ограничение глубины вложенности**: JSON-данные с вложенностью более 50 уровней
  отклоняются для предотвращения атак переполнения стека.
- **Защита от загрязнения прототипа**: Ключи `__proto__`, `constructor` и
  `prototype` удаляются из импортированного JSON перед обработкой.
- **Ограничения количества сущностей**: Общее количество сущностей в снимке
  проверяется на соответствие настраиваемым лимитам.
- **Ограничение размера**: Конечная точка импорта принимает данные размером до 50 МБ.
- **Защита CSRF**: Все конечные точки импорта (POST) требуют действительный CSRF-токен
  через заголовок `x-csrf-token`.

## Конечные точки API

| Метод | Путь | Описание |
|-------|------|----------|
| `GET` | `/api/v1/metahub/:id/export` | Экспорт полного метахаба как снимка |
| `POST` | `/api/v1/metahubs/import` | Импорт снимка как нового метахаба |
| `GET` | `/api/v1/metahub/:id/publication/:pubId/versions/:verId/export` | Экспорт версии публикации |
| `POST` | `/api/v1/metahub/:id/publication/:pubId/versions/import` | Импорт снимка как новой версии |

## Устранение неполадок

- **"Hash mismatch"**: Файл мог быть изменён после экспорта. Выполните повторный
  экспорт из источника.
- **"Invalid envelope"**: Импортированный файл не соответствует ожидаемой схеме.
  Убедитесь, что он был экспортирован из совместимой версии Universo Platformo.
- **"Entity limit exceeded"**: Снимок содержит больше сущностей, чем допускает
  настроенный максимум. Обратитесь к администратору для увеличения лимитов.
