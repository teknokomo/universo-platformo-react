# Создание нового приложения

В текущем репозитории создание application обычно затрагивает
метаданные приложения, членство, publication-связи,
контролируемое поведение схем и интеграцию фронтенд- или бэкенд-пакетов.

## Рекомендуемый подход

1. Определите, нужны ли application только управляемые метаданные или собственная схема рантайма.
2. Уточните, как оно связано с metahubs, publications и членством.
3. Добавляйте или расширяйте маршруты бэкенда и SQL-first persistence только при необходимости.
4. Подключайте фронтенд через общую оболочку и пакеты API client.
5. Валидируйте поведение точечными тестами и корневой сборкой.

Рассматривайте создание application как задачу проектирования платформы с общими последствиями для рантайма.
