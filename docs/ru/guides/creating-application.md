# Создание нового приложения

В текущем репозитории создание приложения обычно затрагивает
метаданные приложения, членство, связи с публикациями,
контролируемое поведение схем и интеграцию фронтенд- или бэкенд-пакетов.

![Параметры приложения](../.gitbook/assets/quiz-tutorial/application-settings-general.png)

## Рекомендуемый подход

1. Сначала определите неизменяемые параметры приложения:
   видимость (`closed` или `public`) и включён ли режим рабочих пространств.
2. Если приложение публичное, включайте рабочие пространства сразу, чтобы каждый участник получал изолированные данные.
3. Уточните, как приложение связано с метахабами, публикациями, членством и синхронизацией рантайм-схемы.
4. Переиспользуйте общий паттерн диалогов `General / Parameters`, а не придумывайте отдельный поток формы.
5. Когда рантайм-схема создаётся из публикации, держите подсистему рабочих пространств внутри той же канонической линии снимков.
6. Добавляйте или расширяйте маршруты бэкенда и SQL-first сохранение только при необходимости.
7. Валидируйте поведение точечными тестами и корневой сборкой.

Рассматривайте создание приложения как задачу проектирования платформы с общими последствиями для рантайма.
