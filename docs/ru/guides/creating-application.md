# Создание нового приложения

В текущем репозитории создание application обычно затрагивает
метаданные приложения, членство, publication-связи,
контролируемое поведение схем и интеграцию фронтенд- или бэкенд-пакетов.

## Рекомендуемый подход

1. Сначала определите неизменяемые параметры приложения:
   видимость (`closed` или `public`) и включён ли workspace-режим.
2. Если приложение публичное, включайте workspaces сразу, чтобы каждый участник получал изолированные данные.
3. Уточните, как application связано с metahubs, publications, membership и runtime schema sync.
4. Переиспользуйте общий паттерн диалогов `General / Parameters`, а не придумывайте отдельный form flow.
5. Когда runtime schema создаётся из publication, держите workspace-подсистему внутри того же canonical snapshot lineage.
6. Добавляйте или расширяйте маршруты бэкенда и SQL-first persistence только при необходимости.
7. Валидируйте поведение точечными тестами и корневой сборкой.

Рассматривайте создание application как задачу проектирования платформы с общими последствиями для рантайма.
