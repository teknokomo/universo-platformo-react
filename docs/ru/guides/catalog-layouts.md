# Макеты каталогов

Макеты каталогов являются overlay-макетами, а не fork-копиями.
Каждый макет каталога указывает на базовый общий макет и затем добавляет только отличия конкретного каталога.

## Модель overlay

- унаследованные виджеты остаются связаны с базовым общим макетом;
- виджеты, принадлежащие каталогу, живут только внутри макета каталога;
- точечные переопределения хранят изменения видимости и расположения унаследованных виджетов;
- конфигурация унаследованных виджетов по-прежнему берётся из базового макета.

## Создание первого макета каталога

![Редактор макета с виджетом квиза](../.gitbook/assets/quiz-tutorial/layout-quiz-widget.png)

1. Откройте целевой metahub.
2. Перейдите в рабочее пространство ресурсов -> Макеты для глобального контекста или откройте маршрут каталога, который ведёт в его список макетов.
3. Создайте макет каталога и выберите базовый общий макет.
4. При необходимости переставьте унаследованные виджеты, переключите их активность или добавьте виджеты только для каталога.

## Владение рантайм-поведением

Выбранный макет каталога владеет рантайм-поведением каталога, таким как:
- видимость кнопки создания;
- режим поиска;
- тип поверхности для создания, редактирования и копирования.
Пока первый макет каталога не создан, рантайм продолжает использовать поведение выбранного общего макета как базовое значение по умолчанию.

## Публикация и рантайм

![Коннекторы опубликованного приложения](../.gitbook/assets/platform/application-connectors.png)

Публикация уплощает эффективный макет каталога в обычные рантайм-строки макета и виджета.
Приложения не вычисляют overlay-логику на лету, а используют уже материализованное рантайм-состояние.
Во время материализации унаследованные виджеты сохраняют конфигурацию из базового макета; overlay-строки меняют только расположение и активность.
