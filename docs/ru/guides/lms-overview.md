# Обзор LMS

Текущая LMS-поверхность в Universo Platformo реализована как конфигурация metahub плюс общие runtime-возможности приложений.
Это не захардкоженный вертикальный модуль внутри `packages/apps-template-mui`, а поставляемое demo теперь приходит из канонического двуязычного fixture Orbital Academy.

## Что покрывает MVP

- классы и каталог студентов как обычные сущности metahub;
- учебные модули как runtime-строки со структурированными элементами контента;
- каталоги тестов и фиксацию ответов;
- совместную работу преподавателей и операторов внутри одного приложения с изоляцией по рабочим пространствам;
- публичные ссылки, позволяющие гостю ввести имя, открыть модуль или тест и отправлять прогресс без регистрации;
- генерацию QR-кодов в оболочке приложения для распространения модулей и тестов.

## Что пока вне рамок MVP

- ИИ-тьютор и генерация контента;
- полноценные пакеты аналитики и отчётности;
- сложные правила оценивания, сертификаты и расписания;
- отдельные LMS-only frontend-пакеты вне общего MUI template.

## Базовые строительные блоки

1. Встроенный шаблон metahub `lms` определяет каноническую структуру сущностей: классы, студенты, модули, тесты, access links, прогресс, записи и supporting enumerations.
2. Backend приложений управляет рабочими пространствами и публичной runtime-поверхностью для гостевого доступа.
3. Общий MUI template предоставляет переключение рабочих пространств, рендеринг QR-кодов, просмотр модулей и виджеты статистики.
4. Коммитнутые generator и snapshot contract поставляют richer EN/RU dataset Orbital Academy с несколькими классами, модулями, тестами, seeded progress и двумя guest-access маршрутами.

## Runtime-модель

Аутентифицированные пользователи работают в обычном runtime приложения по адресу `/a/:applicationId`.
Гости используют публичный маршрут `/public/a/:applicationId/links/:slug`, вводят отображаемое имя, получают гостевой session token и продолжают работу без логина в платформе.
Если у приложения включены рабочие пространства, публичный runtime привязывает данные к workspace access link или текущей гостевой сессии, а не к произвольному активному workspace.
Состояние гостевой сессии хранится в session storage текущей вкладки или браузерной сессии, а не в долговечном local storage общего устройства.

## Проверенное browser-покрытие

Поставляемый LMS browser-набор покрывает управление рабочими пространствами, негативные сценарии публичных ссылок, раздачу через QR-код, виджет статистики, EN guest journey и RU guest route с локализованными module, quiz и access-link текстами.
