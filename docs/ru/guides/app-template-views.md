# Настройки отображения шаблона приложения

Дашборд apps-template-mui поддерживает настраиваемые режимы отображения,
которые позволяют конечным пользователям переключаться между табличным и карточным макетами,
фильтровать данные с помощью строки поиска и настраивать параметры отображения строк.

Создание макетов находится в разделе Ресурсы -> Макеты внутри метахаба.
Это руководство использует этот канонический путь навигации.

## Обзор настроек отображения

![Настройки отображения макета](../.gitbook/assets/quiz-tutorial/layout-quiz-widget.png)

Настройки отображения хранятся в JSONB-поле `config` макета метахаба и
применяются во время выполнения дашбордом приложения. Все настройки опциональны —
существующие дашборды продолжают отображаться в стандартном табличном режиме,
когда настройки не заданы.

## Доступные настройки

| Настройка | Тип | По умолчанию | Описание |
|-----------|-----|--------------|----------|
| `showViewToggle` | `boolean` | `false` | Показать переключатель режима карточки/таблица в панели инструментов |
| `defaultViewMode` | `'table' \| 'card'` | `'table'` | Начальный режим отображения, когда у пользователя нет сохранённых предпочтений |
| `showFilterBar` | `boolean` | `false` | Показать поле поиска в панели инструментов |
| `cardColumns` | `2 \| 3 \| 4` | `3` | Количество колонок в сетке карточек |
| `rowHeight` | `number \| 'auto'` | компактный | Высота строки: фиксированное значение в пикселях (мин. 36) или `'auto'` для авторазмера по содержимому |

## Настройка параметров отображения

### Через интерфейс деталей макета

1. Перейдите в метахаб -> Ресурсы -> Макеты.
2. Выберите макет для открытия страницы деталей.
3. Прокрутите до раздела **Настройки отображения приложения**.
4. Переключайте параметры и выбирайте значения для каждой настройки.
5. Изменения сохраняются немедленно в конфигурации макета.

### Через API

Обновите JSONB-поле `config` макета напрямую:

```json
{
  "config": {
    "showViewToggle": true,
    "defaultViewMode": "card",
    "showFilterBar": true,
    "cardColumns": 3,
    "rowHeight": "auto"
  }
}
```

## Переключатель карточки/таблица

Когда `showViewToggle` включён, панель инструментов дашборда отображает кнопки
режима отображения. Пользователи могут переключаться между:

- **Табличный вид**: Стандартное отображение DataGrid с сортируемыми столбцами.
- **Карточный вид**: Адаптивная сетка карточек с краткой информацией об элементах.

Предпочтение пользователя сохраняется в `localStorage` через хук `useViewPreference`
из `@universo/template-mui`.

## Строка поиска/фильтрации

Когда `showFilterBar` включён, в панели инструментов появляется поле поиска.
Поиск выполняет клиентскую фильтрацию по всем видимым текстовым столбцам.
Поиск работает с задержкой (300мс) для предотвращения избыточных перерисовок.

## Параметры высоты строк

Настройка `rowHeight` управляет отображением строк DataGrid:

- **Компактный** (по умолчанию): Стандартная компактная плотность (~36px строки).
- **Нормальный** (например, `52`): Фиксированная высота строки в пикселях.
- **Авто**: Высота по содержимому с переносом текста.

Когда `rowHeight` установлен в `'auto'`, ячейки используют `white-space: normal` и
`word-break: break-word` для многострочного отображения текста.

## Архитектура интеграции

Улучшенная секция дашборда в `MainGrid.tsx` связывает существующий DataGrid
с общими компонентами из `@universo/template-mui`:

- `ViewHeaderMUI` — строка поиска с элементами управления панели инструментов
- `ToolbarControls` — кнопки переключения режима отображения
- `ItemCard` — компонент отдельной карточки для карточного вида
- `PaginationControls` — навигация по страницам с адаптером состояния пагинации
- `useViewPreference` — сохранение режима отображения в localStorage

Улучшение защищено флагом `isEnhancedMode`, который активируется только когда
`showViewToggle` или `showFilterBar` явно установлены в конфигурации макета.
