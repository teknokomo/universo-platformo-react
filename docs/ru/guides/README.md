# Руководства

Эти руководства намеренно сделаны практичными и привязанными к репозиторию. Они объясняют, как работать с текущей
формой платформы и безопасно подходить к типовым задачам.

## Доступные руководства

- [Рабочее пространство ресурсов](general-section.md) — навигация и проектирование общих макетов.
- [Макеты, привязанные к сущностям](entity-scoped-layouts.md) — overlay-проектирование, наследование и владение рантайм-поведением.
- [Настройки отображения шаблона приложения](app-template-views.md) — рантайм-настройки на уровне макета.
- [Создание приложений](creating-application.md) — поток создания приложения и коннектора.
- [Публикация контента](publishing-content.md) — передача релиза из метахаба в приложение.
- [Работа с UPDL](working-with-updl.md) — структурное моделирование с семействами узлов UPDL.
- [Мультиплатформенный экспорт](multi-platform-export.md) — текущие границы переносимости.
- [Скрипты метахаба](metahub-scripting.md) — исполняемые области, общие библиотеки и fail-closed правила проектирования.
- [Обновление схем системных приложений](updating-system-app-schemas.md) — как добавлять, изменять и удалять поля и таблицы в манифестах.

Используйте эти страницы как ориентиры, а затем переходите к разделу
архитектуры или README нужного пакета, когда нужны более низкоуровневые детали реализации.
