# Руководства

Эти руководства намеренно сделаны практичными и привязанными к репозиторию. Они не
пытаются описать все будущие возможности, а объясняют, как безопасно подходить
к типовым задачам в рамках текущей формы платформы.

## Доступные руководства

- [General Section](general-section.md) — навигация и authoring глобальных layout.
- [Catalog Layouts](catalog-layouts.md) — overlay authoring, inheritance и владение runtime behavior.
- [Настройки отображения шаблона приложения](app-template-views.md) — runtime settings на уровне layout.
- Создание нового приложения.
- Публикация контента.
- Работа с UPDL.
- Мультиплатформенный экспорт.
- Metahub Scripting.
- [Обновление схем системных приложений](updating-system-app-schemas.md) — как добавлять, изменять и удалять поля и таблицы в манифестах.

Используйте эти страницы как ориентиры, а затем переходите к разделу
архитектуры или README нужного пакета, когда нужны более низкоуровневые детали реализации.
