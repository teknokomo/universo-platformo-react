# Узлы и рабочие пространства

Узлы и рабочие пространства описывают операционные контуры для организаций,
людей, принадлежащих ресурсов и правил совместной работы по всей платформе.

## Что представляет этот контур

- Границу для людей, ролей, членства и принадлежащих ресурсов.
- Место, к которому можно привязать операционные данные и правила совместной работы.
- Единицу, которая делает решения о совместной работе и доступе явными.

## Текущая реализация

![Управление рабочими пространствами приложения](../.gitbook/assets/platform/applications-list.png)

Текущая публичная реализация уже включает аутентифицированное членство,
инициализацию профиля, админ-роли, доступ к metahubs и привязанный к запросу доступ к данным.

Теперь приложения используют этот контур напрямую:

- приложения с включёнными рабочими пространствами создают персональные рабочие пространства для владельца и каждого участника;
- общий MUI shell приложения показывает видимый переключатель рабочих пространств в навигационной панели приложения;
- пользователь может создавать общее рабочее пространство и управлять участниками, не покидая рантайм-поверхность;
- роли рабочих пространств хранятся отдельно от членства приложения, чтобы правила совместного доступа оставались явными;
- рантайм-строки изолируются по рабочему пространству через backend-предикаты и рантайм-контекст;
- сценарии `join`, `leave`, добавление участника и удаление участника синхронно поддерживают жизненный цикл рабочего пространства.

Репозиторий реализует базовые паттерны координации и контроля доступа.

Актуальный пользовательский контракт описан в [руководстве по управлению рабочими пространствами](../guides/workspace-management.md).
