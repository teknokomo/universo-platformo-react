# Узлы и рабочие пространства

Узлы и рабочие пространства описывают операционные контуры для организаций,
людей, принадлежащих ресурсов и правил совместной работы по всей платформе.

## Что представляет этот контур

- Границу для людей, ролей, членства и принадлежащих ресурсов.
- Место, к которому можно привязать операционные данные и правила совместной работы.
- Единицу, которая позже сможет участвовать в более широкой распределённой координации.

## Текущая реализация

Текущая публичная реализация уже включает аутентифицированное членство,
инициализацию профиля, админ-роли, доступ к metahubs и привязанный к запросу доступ к данным.

Теперь applications используют этот контур напрямую:

- приложения с включёнными workspaces создают персональные рабочие пространства для владельца и каждого участника;
- workspace roles хранятся отдельно от membership приложения, чтобы future-sharing можно было добавить без переделки схемы;
- runtime-строки изолируются по workspace через backend predicates и runtime context;
- сценарии `join`, `leave`, добавление участника и удаление участника синхронно поддерживают lifecycle рабочего пространства.

Точная доменная лексика ещё может уточняться, но базовые паттерны
координации и контроля доступа в репозитории уже реализованы.
