# Рабочее пространство ресурсов

Рабочее пространство ресурсов является метахаб-уровнем проектирования для артефактов, которые должны переиспользоваться сразу в нескольких целевых объектах.
Оно отделено от CRUD отдельных объектов, потому что владеет общим проектным контрактом до того, как публикация развернёт его в runtime-состояние.

![Рабочее пространство ресурсов](../../.gitbook/assets/entities/resources-workspace.png)

## Вкладки

- Макеты: общие макеты и переопределения макетов для конкретных типов сущностей.
- Атрибуты: общие пулы атрибутов для типов сущностей с возможностью `dataSchema`.
- Константы: общие пулы констант для типов сущностей с возможностью `fixedValues`.
- Значения: общие пулы значений для типов сущностей с возможностью `optionValues`.
- Общие скрипты: библиотечные скрипты уровня ресурсов, импортируемые через `@shared/<codename>`.

## Правила работы

- Открывайте рабочее пространство ресурсов, когда артефакт должен сначала стать общим, а затем изменяться только точечными переопределениями для целевых объектов.
- Открывайте целевую сущность, когда нужно проверить итоговый объединённый унаследованный результат.
- Держите исключения и изменения активности точечными, а не клонируйте общие строки.
- Публикуйте и синхронизируйте связанное приложение, чтобы проверить runtime-разворачивание общего артефакта.

## Результат в runtime

Общие строки остаются обычными проектными строками внутри своих переиспользуемых пулов ресурсов, но публикация разворачивает их в обычные runtime-метаданные связанного приложения.
Это сохраняет переиспользуемое проектирование и одновременно удерживает runtime-таблицы плоскими и предсказуемыми.
