# Общие скрипты

Общие скрипты — это библиотечные модули из рабочего пространства ресурсов, которые публикуют переиспользуемые вспомогательные функции для других скриптов метахаба.
Они являются проектными ресурсами только для импорта и не раскрываются как прямые рантайм-точки входа.

![Поверхность проектирования общих скриптов в диалоге скриптов метахаба](../../.gitbook/assets/quiz-tutorial/metahub-scripts.png)

## Правила проектирования

- Создавайте общие скрипты только из вкладки «Скрипты» в рабочем пространстве ресурсов.
- Всегда связывайте `attachedToKind=general` с `moduleRole=library`.
- Держите библиотечный код чистым, а исполняемое поведение выносите в потребляющие скрипты.
- Импортируйте общие вспомогательные функции из потребителей через `@shared/<codename>`.

## Правила отказа по умолчанию

- Общие скрипты отклоняют роли, отличные от `library`.
- Новые библиотечные скрипты отклоняют области привязки, отличные от `general`.
- Удаление и переименование `codename` завершаются ошибкой, пока зависимые потребители ещё импортируют библиотеку.
- Циклические графы `@shared/*` падают до того, как публикация сможет отгрузить рантайм-состояние.

## Публикация и рантайм

Публикация валидирует общие библиотеки в порядке зависимостей до компиляции потребляющих скриптов.
Рантайм оставляет логику общих библиотек доступной только через скомпилированных потребителей вместо показа строк библиотек как исполняемых рантайм-скриптов.
