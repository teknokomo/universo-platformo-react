# Общие модули

Общие модули — это библиотечные модули из рабочего пространства ресурсов, которые публикуют переиспользуемые вспомогательные функции для других модулей метахаба.
Они являются проектными ресурсами только для импорта и не раскрываются как прямые рантайм-точки входа.

![Поверхность проектирования общих модулей в диалоге модулей метахаба](../../.gitbook/assets/quiz-tutorial/metahub-modules.png)

## Правила проектирования

- Создавайте общие модули только из вкладки «Модули» в рабочем пространстве ресурсов.
- Всегда связывайте `attachedToKind=general` с `moduleRole=library`.
- Держите библиотечный код чистым, а исполняемое поведение выносите в потребляющие модули.
- Импортируйте общие вспомогательные функции из потребителей через `@shared/<codename>`.

## Правила отказа по умолчанию

- Общие модули отклоняют роли, отличные от `library`.
- Новые библиотечные модули отклоняют области привязки, отличные от `general`.
- Удаление и переименование `codename` завершаются ошибкой, пока зависимые потребители ещё импортируют библиотеку.
- Циклические графы `@shared/*` падают до того, как публикация сможет отгрузить рантайм-состояние.

## Публикация и рантайм

Публикация валидирует общие библиотеки в порядке зависимостей до компиляции потребляющих модулей.
Рантайм оставляет логику общих библиотек доступной только через скомпилированных потребителей вместо показа строк библиотек как исполняемых рантайм-модулей.
