# Исключения

Исключения — это точечные переопределения для целевых объектов, а не клонированные копии общих строк.
Они позволяют одному общему источнику оставаться центральным, пока выбранные объекты отказываются от наследования.

![Список общих атрибутов, где настраивается унаследованное поведение](../../.gitbook/assets/entities/shared-attributes.png)

## Как это работает

- Общая исходная строка остаётся в общем пуле рабочего пространства ресурсов.
- Строки переопределений для целевых объектов записывают различия по исключению, активности и порядку.
- Очистка переопределения возвращает объект к унаследованному поведению по умолчанию.
- Списки целевых объектов читают объединённый результат вместо редактирования общего источника на месте.

## Правила работы

- Настраивайте исключения из рабочего пространства ресурсов во время редактирования общей строки, а не из унаследованной строки целевого объекта.
- Используйте исключения только тогда, когда `sharedBehavior.canExclude` всё ещё разрешает отказаться от наследования.
- Оставляйте локальные сущности локальными вместо создания общей строки с исключением для всех.
- Проверяйте включённые и исключённые целевые объекты перед публикацией, когда общая строка влияет на критичное рантайм-поведение.
