# Узлы компонентов

Component nodes добавляют другим nodes переиспользуемые возможности, поля,
поведения или metadata.

## Типичные обязанности

- Добавлять optional- или repeatable-возможности к базовой entity.
- Держать переиспользуемые concerns отдельно от первичной identity.
- Поддерживать композицию вместо глубокой иерархии наследования.

С точки зрения платформы component nodes могут выражать бизнес-поля,
настройки представления, интеграционную metadata или execution-возможности.
