# Шаблон метахаба «PlayCanvas»

`PlayCanvas` — это opt-in шаблон метахаба, который включает новый тип сущности `Проекты` вместе с базовыми пресетами. Это рекомендуемая стартовая точка для метахабов, авторский контент которых идёт через PlayCanvas Editor и runtime.

Шаблон не выбирается по умолчанию. Новые метахабы по-прежнему стартуют с `Basic`, пока пользователь явно не выберет `PlayCanvas`.

## Каталог пресетов

Шаблон включает следующие пресеты:

| Пресет             | Роль                                            |
| ------------------ | ----------------------------------------------- |
| `project`          | Связывает каждый проект PlayCanvas Editor 1:1 |
| `hub`              | Верхний уровень иерархии (без изменений)        |
| `page`             | Структурированный блочный контент (без изменений) |
| `object`           | Записи/компоненты (без изменений)              |
| `set`              | Типизированные константы (без изменений)         |
| `enumeration`      | Закрытые списки значений (без изменений)          |

Секция `Проекты` помещается над `Хабами` в левом меню через `sidebarOrder: 5`.

## Чего шаблон не делает

Шаблон **не** декларирует подключение пакетов метахаба. Подключение `@universo-react/playcanvas-editor-frontend`, `@universo-react/playcanvas-engine`, `@universo-react/colyseus-client` и `@universo-react/colyseus-server` — это runtime-шаг в метахабе (панель «Ресурсы → Пакеты»), как и сейчас. Это сохраняет манифест шаблона простым (только `presets` + `seed`) и не привязывает версионирование шаблона к каталогу пакетов.

## Пример

Генератор MMOOMM (Universo MMO) разворачивает метахаб из шаблона `PlayCanvas`, затем создаёт:

1. Экземпляр `Проекты` с именем `MMOOMM Authoring`.
2. Запись `Project`, созданная и связанная с этим экземпляром (`config.projectBinding`).
3. Flight, ship, station, команды движения и константы симуляции — в стандартных пресетах object / set / enumeration.
4. Runtime-модули (виджет полётной сцены, fixed-tick flight runtime) и опубликованное приложение.

Сгенерированный снапшот коммитится в `tools/fixtures/metahubs-mmoomm-app-snapshot.json`.
