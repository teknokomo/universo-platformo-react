# Guides

These guides are intentionally practical and repository-aware. They do not try
to document every future capability; they explain how to think about the
current platform shape and how to approach common tasks safely.

## Available Guides

- Creating a New Application.
- Publishing Content.
- Working with UPDL.
- Multi-Platform Export.
- [Updating System App Schemas](updating-system-app-schemas.md) — how to add, change, or remove fields and tables in fixed system app manifests.

Use these pages as orientation guides, then move into architecture or package
READMEs when you need lower-level implementation details.
